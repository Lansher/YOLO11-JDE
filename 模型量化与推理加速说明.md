# YOLO11-JDE 权重：量化、导出与测速说明

本文说明如何对权重文件 `YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt`（JDE 联合检测+Re-ID，输入约 **1280** 边长）做 **FP16 / INT8 量化**、导出到 **ONNX / TensorRT / OpenVINO / NCNN** 等格式，并测量 **单帧延迟（ms）** 与 **FPS**，用于填写实验表格。

---

## 1. 环境与路径

在克隆的 **YOLO11-JDE** 仓库根目录下操作（保证使用本仓库自带的 `ultralytics`，而不是 pip 里另一套版本）：

```bash
cd /path/to/YOLO11-JDE
pip install -e .
# 导出常用依赖（按需安装）
pip install onnx onnxruntime onnxruntime-gpu  # ONNX / 测速
# TensorRT：需在装有 NVIDIA 驱动与 CUDA 的机器上，由 NVIDIA 提供 tensorrt 包
# OpenVINO：pip install openvino-dev
```

**加载权重时必须指定任务为 JDE**，与 `scripts/track.py` 中用法一致：

```python
from ultralytics import YOLO

WEIGHTS = "YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt"
model = YOLO(WEIGHTS, task="jde")
```

下文默认 `imgsz=1280`（若你训练时使用非正方形输入，按实际 `imgsz` 填写）。

---

## 2. 表格列与推荐填法

| 部署框架 / 模型格式 | 数据精度 | 如何得到 |
|---------------------|----------|----------|
| PyTorch（`.pt` 原生） | FP32 | 默认 `model.predict` / `model.track`，不做 `half` |
| PyTorch | FP16 | GPU 上 `half=True`，或 `model.model.half()` 后推理 |
| ONNX（`.onnx`） | FP32 | `export(format="onnx", half=False)` |
| ONNX | FP16 | GPU 上 `export(format="onnx", half=True, device=0)`（见官方对 half 与 device 的限制） |
| TensorRT（`.engine`） | FP16 | `export(format="engine", half=True, device=0)` |
| TensorRT | INT8 | `export(format="engine", int8=True, data=你的yaml, device=0)`，需校准数据 |
| OpenVINO | FP16 / INT8 | `export(format="openvino", half=True/False, int8=True/False, data=...)` |
| NCNN | 一般为 FP16/INT8 组合 | `export(format="ncnn", ...)` |

**注意**：INT8 会改变特征分布，**可能影响 Re-ID 分支质量**；填表前建议在验证集上对比 MOTA/IDF1 或特征检索指标，而不仅看 FPS。

---

## 3. 导出命令（CLI）

在仓库根目录执行，将路径换成你的 `.pt` 实际位置：

```bash
# ONNX FP32，固定 batch 与尺寸（表格测速建议固定 shape，dynamic=False）
yolo export model=YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt task=jde \
  format=onnx imgsz=1280 batch=1 dynamic=False simplify=True

# ONNX FP16（需在 GPU 上导出）
yolo export model=YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt task=jde \
  format=onnx imgsz=1280 batch=1 half=True device=0 dynamic=False simplify=True

# TensorRT FP16（需要 GPU + TensorRT）
yolo export model=YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt task=jde \
  format=engine imgsz=1280 batch=1 half=True device=0 dynamic=False

# TensorRT INT8（必须提供 data 做校准；可换成你自己的 data yaml）
yolo export model=YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt task=jde \
  format=engine imgsz=1280 batch=1 int8=True device=0 dynamic=True \
  data=/path/to/your_dataset.yaml

# OpenVINO（CPU 常用）
yolo export model=YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt task=jde \
  format=openvino imgsz=1280 half=True
# INT8 时通常需要 int8=True 且指定 data
```

## 4. 导出命令（Python API）

```python

```

成功后会打印保存路径（如 `.onnx`、`.engine`、`*_openvino_model/`）。

## 5. 测速：Ultralytics 内置 benchmark（多格式对比）

对 **PyTorch 权重** 可一键对比多种导出格式的速度与精度（会尝试导出多种格式，环境需齐全）：

```python
from ultralytics import YOLO

model = YOLO("YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt", task="jde")
df = model.benchmark(data="coco8.yaml", imgsz=1280, half=False, int8=False, device="0", verbose=True)
print(df)
```

- `data`：INT8 相关或验证时会用到；可换成你自己的 `yaml`。
- 若某格式在当前机器不可用，对应行会失败，属正常现象。

## 6. 测速：手写循环（推荐用于「填表」的统一口径）

口径建议写清楚：**仅网络前向** 还是 **含 track 关联**；输入是否为固定 `1280`；batch=1；GPU 上需 `torch.cuda.synchronize()`。

### 6.1 PyTorch FP32 / FP16（单帧）

```python
import time
import torch
from ultralytics import YOLO

weights = "YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt"
model = YOLO(weights, task="jde")
# FP16：half=True
# model = YOLO(weights, task="jde")
# ...

img = "ultralytics/assets/bus.jpg"  # 换成你的测试图
n_warmup, n_iter = 50, 200

for _ in range(n_warmup):
    model.predict(img, imgsz=1280, half=False, verbose=False)

t0 = time.perf_counter()
for _ in range(n_iter):
    model.predict(img, imgsz=1280, half=False, verbose=False)
if torch.cuda.is_available():
    torch.cuda.synchronize()
t1 = time.perf_counter()

ms = (t1 - t0) / n_iter * 1000
fps = n_iter / (t1 - t0)
print(f"mean latency: {ms:.3f} ms, FPS: {fps:.2f}")
```

将 `half=True` 可测 FP16（需 CUDA）。

### 6.2 已导出 ONNX（ONNX Runtime）

```python
import time
import numpy as np
import onnxruntime as ort

session = ort.InferenceSession(
    "YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.onnx",
    providers=["CUDAExecutionProvider", "CPUExecutionProvider"],
)
inp = session.get_inputs()[0].name
x = np.random.randn(1, 3, 1280, 1280).astype(np.float32)

for _ in range(50):
    session.run(None, {inp: x})

t0 = time.perf_counter()
for _ in range(200):
    session.run(None, {inp: x})
t1 = time.perf_counter()
print((t1 - t0) / 200 * 1000, "ms")
```

（若模型输入名不是默认，以 `session.get_inputs()` 为准。）

### 6.3 TensorRT `.engine`

导出得到 `.engine` 后，仍可用 `YOLO("xxx.engine")` 加载并 `predict` 测速，与上节 PyTorch 测速方式类似；或使用 TensorRT Python/C++ API 只跑推理节点。

## 7. 填表时的建议

1. **固定硬件与驱动**：同一张 GPU、同一 CUDA/TensorRT 版本，结果才可横向比较。  
2. **固定分辨率与 batch**：与你的训练/部署一致（本文以 **1280** 为例）。  
3. **预热**：至少数十次前向再计时不计预热。  
4. **端到端跟踪**：若表格要的是「整条跟踪管线 FPS」，请用 `model.track(..., imgsz=1280)` 在视频上计时，通常会低于纯检测网络 FPS。  
5. **INT8**：务必记录校准集（`data=`）与是否接受精度损失；JDE 建议额外做跟踪指标验证。

---

## 8. 常见问题

- **`half=True` 导出 ONNX 报错或警告**：Ultralytics 对 half ONNX 要求 GPU 导出；请使用 `device=0`。  
- **TensorRT 导出失败**：确认本机已安装与 CUDA 匹配的 TensorRT，并在 GPU 上导出。  
- **INT8 缺少 data**：导出器会使用默认 `data`（JDE 任务下默认类似 `coco8.yaml`），领域差异大时请显式传入你自己的 yaml。  

更通用的部署选项也可参考 Ultralytics 文档：[Model Export](https://docs.ultralytics.com/modes/export/)。
