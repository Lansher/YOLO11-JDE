# 附录：会话级脚本与仓库维护备忘

本页由 `对话总结_2026-03-27.md` 整理而来，记录**特定日期会话**中的产物与排错要点；**排障与 Jetson 修复的权威说明**仍以 [问题修复记录.md](./问题修复记录.md) 为准。

---

## 1. 视频跟踪（`scripts/track.py`）

- 本地权重、`save=True`、输出目录示例：`video_outputs/track/`  
- **`JDEModel` 反序列化失败**：环境中 pip 版 `ultralytics` 与仓库 JDE 结构不一致 → 在脚本中 **`sys.path` 优先本仓库根目录**，使用仓库内 `ultralytics`。  
- **`ModuleNotFoundError: lap`**：安装跟踪关联依赖（如 `lap`）后重试。

---

## 2. 热力图（轨迹累积）

- 脚本：`scripts/heatmap_video.py`  
- 语义：类似 `solutions.Heatmap` 的**时序累积热度**（框出现频率），非 Grad-CAM。  
- `task="jde"`；无 GUI 时对 `cv2.destroyAllWindows()` 做条件调用或保护。

---

## 3. Grad-CAM

- `scripts/gradcam_video.py`：视频叠加；默认 `target-layer=16`、`score-index=4`（视模型结构调整）。  
- `scripts/gradcam_image.py`：单图。  
- `JDEModel` 前向示例形态：`pred` 可为 `[1, C, N]`，另有多尺度特征列表；末层为 `JDE` 头。

---

## 4. `.gitignore` 与大文件

曾追加示例规则：大视频目录、`video_outputs/`、`*.mp4`/`*.avi`/bag 等；权重 `*.pt`、`*.onnx` 等按仓库原有策略。

---

## 5. Git 备忘

- **403 push**：检查 `git remote -v`，确认 `origin` 为有权推送的仓库 URL。  
- **fetch vs push**：fetch 只更新远端引用；push 上传本地提交。

---

- `video_outputs/track/*.avi`  
- `video_outputs/heatmap_*.avi`  
- `video_outputs/gradcam_*.avi` / `gradcam_*.png`  

具体以当前脚本默认参数为准。
