# YOLO11-JDE：部署导出与测速

本文合并整理原《YOLO11s_JDE_量化加速说明》与《模型量化与推理加速说明》，并补充与 **Jetson** 相关的交叉引用。

---

## 1. 文档目的

将权重（如 `YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt`）部署为 **ONNX / TensorRT** 等格式，通过 **FP16** 或 **INT8** 降低延迟；并统一 **测速口径**，便于填实验表格。

典型路径：**PyTorch FP32 → ONNX FP32 → TensorRT FP32 → TensorRT FP16**；需进一步压缩算力时再试 **TensorRT INT8**（需校准数据）。

> **JDE 特性**：除检测外还输出 **ReID 嵌入**。INT8 对嵌入与跟踪的影响通常大于 FP16；务必用 **MOTA / IDF1** 等与 FP16 对比，而非仅看检测或 FPS。

**边缘设备（Jetson）**：PyPI 常无 `onnxruntime-gpu`（aarch64）、torch/torchvision 组合特殊、NMS 与 IR 版本等见 [问题修复记录.md](./问题修复记录.md)。

---

## 2. 环境与路径

在克隆的 **YOLO11-JDE** 仓库根目录操作，保证使用本仓库内 `ultralytics`（`pip install -e .`），避免与 pip 全局旧版混淆。

| 项目 | 说明 |
|------|------|
| Python | 与工程一致 |
| PyTorch + CUDA | TensorRT 导出与 FP16/INT8 引擎需在 **NVIDIA GPU** 上完成（Jetson 见 JP 文档选用 **cp38** 等匹配 wheel） |
| 导出依赖 | `onnx`、`onnx` IR；桌面 GPU 可 `onnxruntime-gpu`；**aarch64** 多见仅 CPU EP |
| TensorRT | 与 CUDA / 驱动匹配 |

加载与导出 **必须** 指定 JDE 任务；`imgsz` 与训练一致（文件名含 `1280px` 时多为 `1280`）：

```python
from ultralytics import YOLO

model = YOLO("YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt", task="jde")
```

推理 `.onnx` / `.engine` 时同样应 **`task="jde"`**（否则易按 detect 后处理，`nc` 误判、候选暴涨、NMS 极慢）。本仓库 `scripts/benchmark_deploy.py` 已对齐。

---

## 3. 推荐实施顺序

1. **基线**：`.pt` FP32 `predict`，记延迟与 FPS。  
2. **优先**：**TensorRT + FP16**（实现简单，精度通常可接受）。  
3. **可选**：**TensorRT + INT8**（`data=xxx.yaml` 校准，回归跟踪）。  
4. **中间验证**：**ONNX**（ONNX Runtime 或再转 TensorRT）。  

---

## 4. 表格列与格式（填表参考）

| 部署 / 格式 | 精度 | 典型做法 |
|-------------|------|----------|
| PyTorch `.pt` | FP32 | `predict` / `track`，`half=False` |
| PyTorch | FP16 | GPU 上 `half=True` |
| ONNX | FP32 | `export(format="onnx", half=False, ...)` |
| ONNX | FP16 | 多在 GPU 上 `half=True, device=0`（以当前导出器限制为准） |
| TensorRT `.engine` | FP16 | `format="engine", half=True` |
| TensorRT | INT8 | `int8=True, data=校准 yaml` |
| OpenVINO / NCNN 等 | 视导出选项 | 见 Ultralytics `export` 文档 |

## 5. 导出：命令行（CLI）

仓库根目录执行，路径换为实际 `.pt`：

```bash
# ONNX FP32（测速建议 batch=1、dynamic=False，shape 固定）
yolo export model=YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt task=jde \
  format=onnx imgsz=1280 batch=1 dynamic=False simplify=True

# ONNX FP16（需 GPU 导出时加 device）
yolo export model=YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt task=jde \
  format=onnx imgsz=1280 batch=1 half=True device=0 dynamic=False simplify=True

# TensorRT FP16
yolo export model=YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt task=jde \
  format=engine imgsz=1280 batch=1 half=True device=0 dynamic=False

# TensorRT INT8（必须 data；(dynamic 等以日志为准）
yolo export model=YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt task=jde \
  format=engine imgsz=1280 batch=1 int8=True device=0 dynamic=True \
  data=/path/to/your_dataset.yaml
```

## 7. 测速

### 7.1 本仓库脚本（与「四行填表」一致）

```bash
python scripts/benchmark_deploy.py --modes all
```

支持 `pt_fp32` / `onnx_fp32` / `trt_fp32` / `trt_fp16` 等；详见脚本内参数。

### 7.2 Ultralytics `model.benchmark`（多格式一键尝试）

```python
from ultralytics import YOLO

model = YOLO("YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt", task="jde")
df = model.benchmark(data="coco8.yaml", imgsz=1280, half=False, int8=False, device="0", verbose=True)
```

某格式在当前环境不可用会失败，属正常。

### 7.3 手写循环（统一口径）

- 写清：**仅网络**还是 **含 NMS/前后处理**；**batch=1**；预热数十次；GPU 上 `torch.cuda.synchronize()`。  
- 与 **跟踪**对标时用 `model.track(...)` 在视频上计 FPS，通常低于纯 `predict`。

示例（FP32 `predict`）：

```python
import time
import torch
from ultralytics import YOLO

model = YOLO("YOLO11s_JDE-CHMOT17-64b-100e_TBHS_m075_1280px.pt", task="jde")
img = "path/to/test.jpg"
n_warmup, n_iter = 50, 200

for _ in range(n_warmup):
    model.predict(img, imgsz=1280, half=False, verbose=False)

t0 = time.perf_counter()
for _ in range(n_iter):
    model.predict(img, imgsz=1280, half=False, verbose=False)
if torch.cuda.is_available():
    torch.cuda.synchronize()
t1 = time.perf_counter()
print(f"mean {(t1-t0)/n_iter*1000:.3f} ms, FPS {n_iter/(t1-t0):.2f}")
```

纯 ONNX Runtime 测「仅 session.run」时，请勿与端到端 `YOLO.predict` 混比；_provider 在 aarch64 上常为 CPU。

## 9. 验收清单

| 项目 | 内容 |
|------|------|
| 正确性 | 同段视频/图上 FP32 / FP16 / INT8 检测与 ID 可接受 |
| 速度 | 固定分辨率与 batch，预热后平均延迟与 FPS |
| 嵌入/跟踪 | INT8 若 ID 切换增多，优先调校准集或退回 FP16 |
| 环境 | CUDA / TensorRT / 驱动版本固定 |

## 10. 常见问题

| 现象 | 方向 |
|------|------|
| 导出失败 | TensorRT/CUDA 版本对齐，或用官方容器 |
| INT8 缺校准 | 提供 `data=` |
| 提速不明显 | 预处理/后处理/IO 占比高；拆分纯推理耗时 |
| 跟踪变差 | 量化伤 ReID；优先 FP16，INT8 用场景数据校准 |
| ONNX 推理 NMS 超时 / 候选爆炸 | **`task="jde"`** 与正确 `nc`；见 [问题修复记录.md](./问题修复记录.md) |
